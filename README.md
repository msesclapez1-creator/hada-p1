Pregunta P1 - ¿Que rama se ha subido?
-  Se ha subido la rama master, que es la rama principal del repositorio.

Pregunta P2 - ¿Qué pasa al utilizar la opción Comparar con la versión no modificada?  
-  Nos muestra las diferencias entre una versión y la otra.

Pregunta P3 - ¿Y ahora ves algún cambio en el repositorio remoto? ¿Por qué? 
-  No se ven lo cambios en el repositorio remoto hasta que haces un git push para enviar los cambios ya que no se suben autonaticamente

Pregunta P4 - ¿Qué significa “Confirmar todo” y cuál sería el comando git equivalente?  
-  Significa que hacemos un commit de todos los cambios y su comando equivalente es: git add. / git commit -m "Actualizacion"

Pregunta P5 - ¿Qué significa “Confirmar todo e insertar” y cuál sería el comando git equivalente?
-  Hacemos un commit local y luego hacemos push al remoto. Comando equivalente: git add. / git commit -m "Actualizacion" / git push

Pregunta P6 - ¿Qué significa “Confirmar todo y sincronizar” cuál sería el comando git equivalente? 
-  Nos sirve para fusionaar los cambios del repositorio remoto con el local y su comando equivalente es: git add. / git commit -m "Actualizacion" / git pull / git push

Pregunta P7 - ¿Cuál es la diferencia entre pulsar Sincronizar o Extraer? ¿Cuáles serían los comandos git equivalentes? 
-  Extraer nos sirve para traer los cambios del remoto al local y su comando es git pull.
-  Mientras que sincronzar nos trae los cambios del remoto y luego envia los del local. Primero hace un git pull y despues un git push
